# Backend Migrations

## When to use this skill:

- Creating new migration files for database schema changes
- Modifying existing tables, columns, indexes, or constraints
- Writing up and down migration scripts
- Managing database versioning and migration history
- Implementing rollback strategies for failed migrations
- Handling data transformations during schema updates
- Creating seed data or initial migration setups
- Working with migration tools like Prisma, Knex, TypeORM, Django migrations, etc.
- Managing foreign key relationships and dependencies in migrations

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle backend migrations.
