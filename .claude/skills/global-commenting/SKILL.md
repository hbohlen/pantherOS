# Global Commenting

## When to use this skill:

- Adding inline comments to explain complex business logic
- Writing JSDoc comments for JavaScript/TypeScript functions and classes
- Creating docstrings in Python, Go, or Rust
- Documenting API endpoints with comments
- Explaining non-obvious code decisions or workarounds
- Writing block comments for complex algorithms
- Adding TODO, FIXME, or NOTE comments for future improvements
- Creating header comments for files explaining purpose
- Writing comments that explain "why" not just "what"
- Adding comments to regular expressions and complex data structures
- Documenting configuration files with inline comments

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global commenting.
