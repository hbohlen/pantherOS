# Global Conventions

## When to use this skill:

- Applying SOLID principles in object-oriented design
- Implementing design patterns (Factory, Observer, Strategy, etc.)
- Following DRY (Don't Repeat Yourself) and KISS principles
- Organizing code into modules, packages, or namespaces
- Creating consistent project directory structures
- Implementing separation of concerns and single responsibility
- Writing clean code with meaningful variable and function names
- Applying MVC, MVP, MVVM, or other architectural patterns
- Creating reusable abstractions and avoiding tight coupling
- Following RESTful conventions and resource naming
- Implementing proper error handling and logging conventions

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global conventions.
