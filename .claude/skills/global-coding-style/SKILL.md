# Global Coding Style

## When to use this skill:

- Writing or editing code in JavaScript, TypeScript, Python, Go, Rust, or any programming language
- Configuring ESLint, Prettier, Black, RuboCop, or other linters and formatters
- Applying consistent naming conventions (camelCase, snake_case, PascalCase, kebab-case)
- Organizing imports and requires statements
- Setting up code formatting rules and style guides
- Creating or updating editorconfig files
- Configuring Prettier, Black, or other code formatters
- Following language-specific style guides (PEP 8 for Python, StandardJS, etc.)
- Maintaining consistent file and folder naming conventions
- Setting up pre-commit hooks for code formatting
- Writing consistent function and class structures across codebases

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global coding style.
