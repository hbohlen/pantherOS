# Global Tech Stack

## When to use this skill:

- Documenting the technology stack in README or architecture docs
- Making decisions about which frameworks or libraries to use
- Setting up package.json, requirements.txt, go.mod, or Cargo.toml
- Configuring development, staging, and production environments
- Selecting databases (PostgreSQL, MongoDB, Redis, etc.)
- Choosing build tools and bundlers (Webpack, Vite, esbuild)
- Setting up CI/CD tools and deployment strategies
- Documenting API requirements and specifications
- Configuring monitoring and logging tools
- Setting up testing frameworks and tools
- Managing version compatibility and upgrade paths

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global tech stack.
